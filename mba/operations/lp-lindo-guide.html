שלב 1 · דקה אחת

הרעיון ב-30 שניות

בעיה ליניארית = למקסם רווח תחת מגבלות. יש לך מוצרים, לכל אחד רווח, ויש משאבים מוגבלים (כסף, נפח, שעות). המחשב מוצא את השילוב הכי רווחי.

הסוד של המבחן: אתה לא מריץ Lindo. הפלט כבר מודפס בשאלון. כל העבודה שלך היא לקרוא נכון את הפלט ולענות על "מה יקרה אם...". זו קריאת מפה, לא תכנות.
שלב 2 · ניסוח

ניסוח בעיה = 3 חלקים, תמיד

① משתני החלטה — מה אני מחליט?

תמיד "כמה מכל דבר": x₁ = כמה תחפושות, x₂ = כמה דגלי ישראל... (המבחן בדרך כלל כבר מגדיר אותם בשבילך).

② פונקציית מטרה — מה ממקסמים?

סכום של (רווח ליחידה × כמות). והנה המלכודת הכי נפוצה:

רווח נטו! המקדם הוא מחיר מכירה − עלות, לא מחיר המכירה.
תשפ״ג: תחפושת נמכרת ב-70 ועולה 50 → המקדם הוא 20:
Max Z = (70−50)x₁ + (15−10)x₂ + (20−15)x₃ + (16,000−10,000)x₄ = 20x₁ + 5x₂ + 5x₃ + 6,000x₄

③ אילוצים — מה מגביל אותי?

  • משאב: תקציב, נפח, שעות — למשל 50x₁+10x₂+15x₃+10,000x₄ ≤ 60,000
  • שוק/מלאי: x₄ ≤ 5 (יש רק 5 שעוני רולקס)
  • שוויון (=): לפעמים חייבים בדיוק — בהתפלה (תשפ״ב): x₁+x₂+x₃ = 20
  • תמיד בסוף: x ≥ 0, ולרוב גם שלמים (Integers)
שלב 3 · הלב של הנושא

לקרוא Lindo — 6 מונחים וגמרנו

כל פלט Lindo בעולם מורכב מאותם 6 דברים. הבן אותם פעם אחת — וכל מבחן ייראה אותו דבר.

OBJECTIVE FUNCTION VALUE
💰 השורה התחתונה

הרווח הכולל של הפתרון האופטימלי. זו התשובה ל"מהו הרווח?".

VALUE
📦 ההרכב הפותח

כמה מכל משתנה בפתרון. VALUE=0 → המוצר הזה בכלל לא נרכש.

REDUCED COST
🪑 שחקן ספסל

רלוונטי רק למשתנה שקיבל 0. אומר בכמה הרווח ליחידה שלו צריך להשתפר כדי שיהיה שווה להכניס אותו למגרש.

SLACK / SURPLUS
🍽️ מה נשאר בצלחת

כמה מהמשאב לא נוצל. SLACK=0 → המשאב נגמר → אילוץ קוֹשֵׁר.

DUAL PRICE
🏷️ מחיר צל

כמה שווה לך עוד יחידה אחת מהמשאב. עוד מ״ק במכולה = עוד 5,333 ₪ רווח. משאב שנשאר ממנו (SLACK>0) → מחיר צל 0.

RANGES
🛡️ תעודת אחריות

הטווח שבו "חוקי המשחק" (הבסיס) לא משתנים. בתוך הטווח — מותר להשתמש במחיר הצל. מחוצה לו — צריך לפתור מחדש.

🔑 4 כללי הזהב (לשנן!)

  1. VALUE > 0 ⟺ REDUCED COST = 0 — מי שמשחק, לא יושב על הספסל.
  2. SLACK > 0 ⟺ DUAL PRICE = 0 — משאב שלא נגמר לא שווה כלום בשוליים.
  3. ROW 2 בפלט = אילוץ (1) בניסוח! Lindo סופר את פונקציית המטרה כשורה 1. תמיד להוריד 1.
  4. מחיר צל תקף רק בתוך טווח ה-RHS. חרגת מהטווח? אסור להשתמש בו.

עכשיו תרגול חי — הפלט האמיתי ממבחן תשע״ו (אלי קומבינה והמכולה)

שלב 4 · 90% מהנקודות

5 תבניות השאלה שחוזרות בכל מועד

צופי ממחזר. אותן 5 שאלות, מספרים אחרים. זהה את התבנית → הפעל את המתכון.

1 "הסבר את הפתרון המתקבל" ◀

🔎 איך מזהים

סעיף א׳ בכל מבחן, 3–4%. "כמה מכל סוג? מהו הרווח?"

🧭 המתכון

מעתיקים את עמודת VALUE ואת OBJECTIVE FUNCTION VALUE. נקודות מתנה — לא לחפור.

📄 תשע״ו סעיף א׳ אלי יביא 20 מסכים, 0 מקררים, 175 מצלמות ו-60 מחשבים. הרווח: 712,000 ₪. זהו.
2 נוסף אילוץ חדש ◀

🔎 איך מזהים

"מתברר כי המכולה מוגבלת ל-2.5 טון" / "לפחות 80% חייבים להיות מי ים" — מגבלה שלא הייתה בניסוח.

🧭 המתכון

מציבים את הפתרון הקיים באילוץ החדש. עומד בו? הפתרון נשאר אופטימלי — סיימת בשתי שורות. לא עומד? הבסיס ישתנה והרווח יקטן (או יישאר).

📄 תשע״ו סעיף ב׳ — מגבלת משקל 2,500 ק״ג
50·20 + 150·0 + 4·175 + 5·60 = 2,000 ≤ 2,500 ✓
הפתרון עומד באילוץ → נשאר אופטימלי, הרווח לא משתנה. סוף.
3 שינוי מחיר / עלות (מקדם מטרה) ◀

🔎 איך מזהים

"מה צריך להיות מחיר X כדי ש..." / "העלות עלתה ל-Y, האם הפתרון משתנה?"

🧭 המתכון

  • משתנה שבפתרון: משווים את השינוי ל-ALLOWABLE INCREASE/DECREASE שלו. בתוך הטווח → כלום לא משתנה (רק Z מתעדכן).
  • משתנה שמחוץ לפתרון (VALUE=0): ה-REDUCED COST הוא בדיוק כמה הרווח שלו צריך לעלות כדי להיכנס.
📄 תשפ״ג סעיף ג׳ — תחפושת השפחה X₁=0 עם REDUCED COST=5. הרווח הנוכחי 20 → צריך רווח של 25 → מחיר מכירה 75 ₪ לפחות (עלות 50).
📄 תשפ״ב סעיף ב׳ — מי הבשור רווח X₃ כרגע 2, ‏ALLOWABLE DECREASE=1 → כשהרווח יורד מתחת ל-1, כלומר עלות מעל 3 ₪, המקור ייצא מהפתרון.
4 עוד / פחות משאב (שינוי RHS) — הכי שווה נקודות ◀

🔎 איך מזהים

"מכולה גדולה יותר" / "המתקן יעבוד רק 700 שעות" / "הלוואה של 5,000 ₪" — צד ימין של אילוץ משתנה.

🧭 המתכון (4 צעדים)

  1. חשב את השינוי Δ = חדש − ישן
  2. בדוק שהוא בתוך טווח ה-RHS (‏ALLOWABLE INCREASE/DECREASE). בפנים? ממשיכים. בחוץ? מחיר הצל לא תקף.
  3. ΔZ = מחיר צל × Δ — זו הפגיעה או התוספת לרווח.
  4. אם צריך כמויות חדשות: פותרים מערכת משוואות של האילוצים הקושרים (דוגמה מלאה בהמשך ⬇).
📄 תשפ״ב סעיף ד׳ — 700 שעות במקום 720 Δ=−20, בדיוק בקצה הטווח (DECREASE=20) → מותר. ‏ΔZ = −20 × 0.1 = −2. הרווח: 32 → 30 (מיליוני ₪).
📄 תשפ״ג סעיף ד׳ — הלוואה 5,000 ₪ בריבית 20% מחיר הצל של התקציב 0.5 → תוספת רווח 5,000×0.5 = 2,500. הריבית עולה 1,000. ‏2,500 − 1,000 = +1,500 → כדאי לקחת!
5 "ויתרנו על חלק מהמוצרים" → פתרון גרפי ◀

🔎 איך מזהים

"אלי מוותר על המקררים והמצלמות" / "רק שני מקורות מים" + המילים "נסח מחדש... פתור בשיטה הגרפית".

🧭 המתכון

נשארים 2 משתנים → כותבים בעיה חדשה וקטנה → משרטטים. המתכון המלא בסעיף "השיטה הגרפית" למטה ⬇.

📄 מופיע בכל שלושת המועדים תשע״ו ד׳ (מסכים+מחשבים), תשפ״ב ה׳ (מי ים+זיקים), תשפ״ג ה׳ (דגלים).
שלב 5 · הדוגמה שמחברת הכל

מכולה של 84 מ״ק — תשע״ו סעיף ג׳, צעד-צעד

השאלה: מציעים לאלי לשדרג מ-60 ל-84 מ״ק. האם כדאי? כמה ירוויח? כמה יקנה מכל מוצר?

  1. השינוי Δ = 84 − 60 = +24 מ״ק. הנפח הוא אילוץ (2) → שורה 3 בפלט.
  2. בדיקת טווח ‏RHS RANGES של שורה 3: ‏ALLOWABLE INCREASE = 52.5. ‏24 < 52.5 → בתוך הטווח, הבסיס נשמר, מחיר הצל תקף.
  3. תוספת הרווח מחיר הצל של הנפח = 5,333.333.
    ΔZ = 24 × 5,333.333 = 128,000 Z חדש = 712,000 + 128,000 = 840,000 ₪
    → כן, כדאי לשדרג.
  4. מי בפתרון? הבסיס נשמר → אותם משתנים: x₁, x₃, x₄ (המקרר x₂ נשאר 0). אבל הכמויות משתנות — יש יותר מקום!
  5. כמויות חדשות — מערכת משוואות 3 אילוצים קושרים ↔ 3 נעלמים. יודעים ש-x₁=20 (אילוץ המלאי קושר), מציבים בשני האחרים:
    תקציב: 1000·20 + 4000x₃ + 2000x₄ = 840,000 → 4000x₃ + 2000x₄ = 820,000 נפח: 0.5·20 + 0.2x₃ + 0.25x₄ = 84 → 0.2x₃ + 0.25x₄ = 74 | ×8000 1600x₃ + 2000x₄ = 592,000 חיסור: 2400x₃ = 228,000 → x₃ = 95 2000x₄ = 820,000 − 4000·95 = 440,000 → x₄ = 220
    תשובה: 20 מסכים, 95 מצלמות, 220 מחשבים. בדיקה: 4500·20+2800·95+2200·220 = 840,000 ✓
למה הסעיף הזה זהב: הוא מתרגל בבת אחת מחיר צל, טווחי RHS, ופתרון משוואות. מי ששולט בו — שולט בתבנית 4 כולה.
שלב 6 · 2 משתנים

השיטה הגרפית — מתכון + הדוגמה מתשע״ו

אלי מוותר על מקררים ומצלמות. נשארו x₁ (מסכים) ו-x₄ (מחשבים):

Max Z = 4500x₁ + 2200x₄ s.t. (1) 1000x₁ + 2000x₄ ≤ 840,000 [תקציב] (2) 0.5x₁ + 0.25x₄ ≤ 60 [נפח] (3) x₁ ≤ 20 [מלאי]

המתכון

  1. כל אילוץ = קו. מוצאים חיתוך עם הצירים: מציבים 0 במשתנה אחד. למשל אילוץ (2): ‏x₁=0→x₄=240; ‏x₄=0→x₁=120.
  2. מסמנים את התחום האפשרי — האזור שמקיים את כל האילוצים יחד (בדרך כלל מצולע קטן ליד הראשית).
  3. מוצאים את נקודות הפינה של המצולע. הפתרון האופטימלי תמיד בפינה.
  4. מציבים כל פינה ב-Z ובוחרים את הגדולה. זהו.
נקודהx₁x₄Z = 4500x₁ + 2200x₄
A000
B20090,000
C ★20200530,000
D0240528,000

מסקנה: 20 מסכים ו-200 מחשבים, רווח 530,000 ₪. שים לב כמה C ו-D קרובות (530 מול 528) — לכן חובה לבדוק את כל הפינות, לא לנחש לפי העין.

שלב 7 · בדוק את עצמך

בוחן עצמי — 4 שאלות של דקה

לאילוץ מסוים SLACK = 3. מה מחיר הצל שלו?
אפס. נשאר עודף מהמשאב — עוד יחידה ממנו לא שווה כלום (כלל זהב 2).
‏X₂: ‏VALUE=0, ‏REDUCED COST=11,916. מציעים להעלות את רווח המקרר ב-10,000. ייכנס לפתרון?
לא. צריך שיפור של לפחות 11,916 ברווח ליחידה. ‏10,000 לא מספיק — המקרר נשאר על הספסל.
מחיר צל של שעות = 0.1, ‏ALLOWABLE INCREASE = 30. מציעים 50 שעות נוספות. מה אפשר לומר?
רק על 30 הראשונות מובטח 0.1 לכל שעה (+3 לרווח). מעבר ל-30 הבסיס משתנה — מחיר הצל כבר לא תקף וצריך לפתור מחדש.
מתקדם: באילוץ שוויון (=20) התקבל מחיר צל −3 (תשפ״ב). מה זה אומר?
כל יחידת דרישה נוספת מקטינה את הרווח ב-3 — כי היא מכריחה לייצר עוד מ״ק במקור היקר (מי ים). מחיר צל שלילי = "המגבלה הזו כבר כואבת לי".
שלב 8 · לפני המבחן

צ׳קליסט טעויות — לעבור עליו ליד הפלט

  • רווח נטו: מקדם המטרה = מחיר − עלות (לא רק מחיר)
  • מספור: ROW 2 בפלט = אילוץ (1) בניסוח
  • מחיר צל — רק בתוך טווח ה-RHS, ורק לאילוץ קושר (SLACK=0)
  • ‏REDUCED COST — רק למשתנה עם VALUE=0
  • בגרפי: לחשב Z לכל כל נקודות הפינה (זוכר את 530 מול 528?)
  • "הסעיפים אינם תלויים זה בזה" — כל סעיף חוזר לפלט המקורי!
  • אם נדרשים שלמים והפתרון לא שלם — לציין זאת בתשובה
